Developing Flutter Applications Inside a DevContainer

Sinan Can Soysal
3 min read
·
Jan 2, 2025

--

In this article, I’ll guide you through setting up a DevContainer for Flutter development. This setup helps streamline your workflow by avoiding local installations and maintaining a clean development environment using Docker and WSL.

Why Use a DevContainer?

Using a DevContainer allows you to:

  • Keep your local machine free from SDKs, libraries, and tools.
  • Ensure consistency across development environments.
  • Easily replicate setups across machines or team members.

Setting Up Your DevContainer

1. Configuring devcontainer.json

Here’s the configuration file for your Flutter DevContainer:

{
    "name": "Flutter Development Container",
    "build": {
        "dockerfile": "Dockerfile"
    },
    "runArgs": [
        "--privileged",
        "-v",
        "/dev/bus/usb:/dev/bus/usb"
    ],
    "forwardPorts": [8081, 5037],
    "remoteUser": "root"
}

This file:

  • Defines the container name and Dockerfile location.
  • Adds USB passthrough for debugging on real devices.
  • Forwards ports 8081 and 5037 for Flutter’s dev server and ADB.

2. Building the Dockerfile

Here’s the Dockerfile to create the container image:

FROM ubuntu:focal

# Set environment variables
ENV ANDROID_SDK_ROOT=/usr/lib/android-sdk

ENV FLUTTER_SDK_ROOT=/usr/lib/flutter
ENV FLUTTER_SDK_VERSION=3.27.1-stable

# Include flutter and android tools in path
ENV PATH="${PATH}:${FLUTTER_SDK_ROOT}/bin:${ANDROID_SDK_ROOT}/cmdline-tools/latest/bin:${ANDROID_SDK_ROOT}/platform-tools"

# Enable noninteractive installation
ENV DEBIAN_FRONTEND=noninteractive

# Install dependencies and tools
RUN apt-get update && apt-get upgrade -y && \
    apt-get install -y \
    watchman \
    openjdk-17-jdk \
    android-sdk \
    wget \
    unzip \
    zip \
    curl \
    git-all \
    xz-utils \
    libglu1-mesa \
    libc6:amd64 libstdc++6:amd64 lib32z1 libbz2-1.0:amd64 && \
    rm -rf /var/lib/apt/lists/*

# Add flutter to safe directory in git
RUN git config --global --add safe.directory /usr/lib/flutter

# Configure git user name and email
RUN git config --global user.name "Your Name" \
    && git config --global user.email "mail@example.com"

# Download and install Android Command Line Tools
RUN mkdir -p ${ANDROID_SDK_ROOT}/cmdline-tools && \
    wget -O commandlinetools.zip "https://dl.google.com/android/repository/commandlinetools-linux-8512546_latest.zip" && \
    unzip commandlinetools.zip -d ${ANDROID_SDK_ROOT}/cmdline-tools && \
    mv ${ANDROID_SDK_ROOT}/cmdline-tools/cmdline-tools ${ANDROID_SDK_ROOT}/cmdline-tools/latest && \
    rm commandlinetools.zip

# Accept Android SDK licenses and install necessary components
RUN yes | ${ANDROID_SDK_ROOT}/cmdline-tools/latest/bin/sdkmanager --licenses && \
    ${ANDROID_SDK_ROOT}/cmdline-tools/latest/bin/sdkmanager "platform-tools" "platforms;android-34" "build-tools;33.0.0"

# Insall Flutter SDK
RUN mkdir -p ${FLUTTER_SDK_ROOT} && \
    cd ${FLUTTER_SDK_ROOT} && \
    curl -OL https://storage.googleapis.com/flutter_infra_release/releases/stable/linux/flutter_linux_${FLUTTER_SDK_VERSION}.tar.xz && \
    tar -xf flutter_linux_${FLUTTER_SDK_VERSION}.tar.xz -C /usr/lib/ && \
    rm -rf flutter_linux_${FLUTTER_SDK_VERSION}.tar.xz

# Disable Flutter telemetry
RUN flutter --disable-analytics

# Default shell to bash
SHELL ["/bin/bash", "-c"]

This setup:

  • Installs Flutter and the Android SDK with essential tools.
  • Configures the PATH for easy access to Flutter and Android tools.

Running the DevContainer

1. Open the DevContainer

- Open Visual Studio Code and navigate to the project folder containing the `devcontainer.json` file.

- Use the Remote — Containers extension to build and open the container.

2. Verify Flutter Installation

Once inside the DevContainer terminal, run the following command to ensure Flutter is properly set up:

flutter doctor

3. Test ADB Setup

If you plan to debug on a real device, check that ADB is functioning, here’s a guide that explains how to connect a device within devcontainers):

adb devices

4. Start Developing

With everything set up, you’re ready to start building and testing your Flutter applications. Your containerized environment ensures a clean and consistent workflow!

Additional Tips

  • Debugging Real Devices: Ensure USB passthrough is configured and use adb devices to confirm device detection.
  • Avoiding Conflicts: Stop any local ADB processes if they’re running on your host machine to prevent port conflicts.
  • Port Forwarding: Modify forwardPorts in devcontainer.json as needed for your application.

Why I Love This Setup

I exclusively use DevContainers with Docker and WSL for all my projects. It keeps my local machine clean and allows me to quickly spin up environments. For Flutter development, this approach ensures a consistent setup and eliminates the hassle of managing tools locally.
